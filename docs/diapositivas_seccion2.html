2.1 — Instalación

¿Cómo se instala Node-RED?

  • 💻
    npm (local) — requiere Node.js instalado. Ideal para desarrollo individual.
    npm install -g --unsafe-perm node-red
  • 🍓
    Raspberry Pi — script oficial que configura Node.js y Node-RED automáticamente.
  • 🐳
    Docker ★ Método del equipo — imagen reproducible, mismo entorno en cualquier SO.
# Levantar el entorno completo en un solo comando
docker compose up --build -d
Ventaja clave: cualquier integrante clona el repo y levanta el entorno idéntico — sin instalar Node.js manualmente.

1.1 — Presentación de la Herramienta

¿Qué es Node-RED?

  • 🧩
    Low-code orientado a eventos — crea soluciones e integraciones conectando nodos.
  • ⚙️
    Nodos = funciones (entrada, transformación, decisiones y comunicación con servicios).
  • 🚀
    Reduce código manual y acelera automatizaciones e integraciones; además, favorece reutilización y ensamblaje visual en flujos.
  • 🌍
    Open source (IBM Emerging Tech Services → OpenJS Foundation). Sitio: nodered.org. Versión estable: 4.1.7.
En resumen: programación visual + integración modular para construir flows desplegables.
1.2 — Arquitectura de la Herramienta

Editor + Runtime

  • 🖥️
    Editor web — diseño visual de flujos con nodos interconectados.
  • 🧠
    Runtime (Node.js) — ejecuta los flows desplegados.
  • 🗂️
    Gestiona flows.json y settings.js (configuración del sistema).
  • 🔗
    Nodos → flows: mensajes se procesan entre funciones e integran APIs, BD y MQTT.
En conjunto: simplicidad visual en el modelado + flexibilidad técnica en la ejecución.
1.3 — Plataforma

Base tecnológica y despliegue

  • 🟨
    Construida sobre Node.js — ejecución orientada a eventos.
  • 🧾
    Lenguaje principal: JavaScript (lógica en nodos tipo function).
  • 🧱
    Multiplataforma: Windows / Linux / macOS y Raspberry Pi.
  • 📦
    Puede desplegarse en local, Docker y nube. Config: settings.js.
  • 🔌
    Extensible con paquetes npm y nodos para integrar: HTTP/REST, MQTT, WebSockets y BD.
En síntesis: plataforma extensible, modular y lista para integración e IoT.
3.1 — Características Generales

Low-code visual para eventos e integración

  • 🧩
    Desarrollo visual: lógica modelada con nodos interconectados (entrada, procesamiento y salidas).
  • 🧱
    Modular y extensible: base de nodos + instalación de paquetes desde su ecosistema.
  • ⚡
    Ejecución flexible sobre Node.js: local, Raspberry Pi, contenedores y nube.
  • 🔁
    Favorece reutilización, automatización e integración entre tecnologías heterogéneas.
  • 🏭
    Alineada con la industrialización del desarrollo: ensamblaje visual de componentes y proceso más ágil.
En pocas palabras: visual, modular y orientada a eventos para construir e integrar rápido.

3.1.2 — Usabilidad

Editor intuitivo y despliegue rápido

  • 🖥️
    Editor visual en navegador: flujos construidos conectando nodos desde una paleta.
  • 🚀
    Deploy con un clic al runtime, lo que acelera prototipado y pruebas.
  • 📦
    Reutiliza componentes y permite lógica en JavaScript cuando se necesita (nodos de función).
  • 📈
    La facilidad disminuye solo si crece la complejidad sin organización, modularización y documentación interna.
Usabilidad alta para integraciones y automatización orientada a eventos; el mantenimiento depende del orden del flujo.
