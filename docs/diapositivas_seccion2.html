Sección 2 — Proceso de Desarrollo

¿Cómo se desarrolla en Node-RED?

El proceso de desarrollo en Node-RED se estructura en torno a la construcción de flujos visuales mediante un editor web. A diferencia de los entornos de desarrollo tradicionales, no requiere compilación ni despliegue manual: los cambios se aplican en tiempo real al presionar el botón Deploy.

Ejemplo de lógica de scripting en un nodo function del módulo ADM

2.1 — Instalación

¿Cómo se instala Node-RED?

  • 💻
    npm (local) — requiere Node.js instalado. Ideal para desarrollo individual.
    npm install -g --unsafe-perm node-red
  • 🍓
    Raspberry Pi — script oficial que configura Node.js y Node-RED automáticamente.
  • 🐳
    Docker ★ Método del equipo — imagen reproducible, mismo entorno en cualquier SO.
# Levantar el entorno completo en un solo comando
docker compose up --build -d
Ventaja clave: cualquier integrante clona el repo y levanta el entorno idéntico — sin instalar Node.js manualmente.

1.1 — Presentación de la Herramienta

¿Qué es Node-RED?

  • 🧩
    Low-code orientado a eventos — crea soluciones e integraciones conectando nodos.
  • ⚙️
    Nodos = funciones (entrada, transformación, decisiones y comunicación con servicios).
  • 🚀
    Reduce código manual y acelera automatizaciones e integraciones; además, favorece reutilización y ensamblaje visual en flujos.
  • 🌍
    Open source (IBM Emerging Tech Services → OpenJS Foundation). Sitio: nodered.org. Versión estable: 4.1.7.
En resumen: programación visual + integración modular para construir flows desplegables.
1.2 — Arquitectura de la Herramienta

Editor + Runtime

  • 🖥️
    Editor web — diseño visual de flujos con nodos interconectados.
  • 🧠
    Runtime (Node.js) — ejecuta los flows desplegados.
  • 🗂️
    Gestiona flows.json y settings.js (configuración del sistema).
  • 🔗
    Nodos → flows: mensajes se procesan entre funciones e integran APIs, BD y MQTT.
En conjunto: simplicidad visual en el modelado + flexibilidad técnica en la ejecución.
1.3 — Plataforma

Base tecnológica y despliegue

  • 🟨
    Construida sobre Node.js — ejecución orientada a eventos.
  • 🧾
    Lenguaje principal: JavaScript (lógica en nodos tipo function).
  • 🧱
    Multiplataforma: Windows / Linux / macOS y Raspberry Pi.
  • 📦
    Puede desplegarse en local, Docker y nube. Config: settings.js.
  • 🔌
    Extensible con paquetes npm y nodos para integrar: HTTP/REST, MQTT, WebSockets y BD.
En síntesis: plataforma extensible, modular y lista para integración e IoT.

Plataformas de despliegue y alcance global

    Despliegue local: Node-RED puede ejecutarse en equipos Windows, Linux y macOS para desarrollo, pruebas o ambientes pequeños.
  • 🍓

  • 🐳

Node-RED no está limitado a un solo entorno; puede pasar de prototipo local a despliegue empresarial con relativa facilidad.

Reflexión final — ¿Para qué sirve Node-RED y para qué no?

Lo que aprendimos usándolo de verdad

✅ Para qué sí sirve

  • ⚡
    Montar integraciones y prototipos rápido — en horas, no días. Ideal cuando necesitas demostrar algo que funciona.
  • 🔌
    Conectar servicios que no hablan entre sí: APIs, bases de datos, MQTT, archivos — sin escribir código de infraestructura.
  • 📊
    Construir dashboards funcionales con formularios, tablas y notificaciones para casos de uso académicos o internos.
  • 🧪
    Validar una idea de negocio o flujo de proceso antes de invertir en una solución más robusta.

❌ Para qué no sirve (o cuesta mucho)

  • 🏗️
    Aplicaciones con lógica de negocio muy compleja — los flows se vuelven difíciles de leer y mantener si no se organizan bien.
  • 💾
    Persistencia real de datos — no tiene base de datos propia. El contexto global se pierde si el contenedor se reinicia.
  • 👥
    Trabajo en equipo sin disciplina — el flows.json genera muchos conflictos en git cuando varios tocan el mismo flow.
🤖 Dificultades con IA (Copilot, ChatGPT)
  • • Las IA sugieren código de versiones antiguas del dashboard — el bug de formValue: [] vs {} lo encontramos porque la IA no conocía el cambio en v2.
  • • Generar flows.json válido es difícil para la IA — los nodos tienen muchas propiedades implícitas que omite.
  • • Hay que verificar todo manualmente: la IA puede sugerir nodos que no existen o paquetes npm deprecados.
